# Web Crawler

A lightweight, **premium‑looking** Python web crawler that starts from a given URL, follows internal links up to a configurable depth, and saves the extracted data (page title, meta description, URL, and outbound links) as JSON.

## Features
- **Domain‑limited** crawling – stays within the start‑page’s domain.
- Configurable **maximum depth** and **request delay**.
- Polite default **User‑Agent** (`AntigravityCrawler/1.0`).
- Graceful error handling – skips pages that cannot be fetched.
- Clean JSON output ready for downstream analysis.

## Quick Start
```bash
# 1. Install dependencies (Python 3.11+ required)
pip install -r requirements.txt

# 2. Run the crawler
python3 crawl.py --start https://example.com --depth 2 --output example_crawl.json
```

Replace `https://example.com` with the site you want to crawl and adjust `--depth` as needed.

## CLI Options
| Option | Description |
|---|---|
| `--start` | Root URL to begin crawling (required). |
| `--depth` | Maximum link depth (default: `1`). |
| `--output` | Path for the JSON result file (default: `crawl_results.json`). |
| `--delay` | Seconds to wait between requests (default: `1.0`). |

## Output Format
The generated JSON is an array of objects, each with:
```json
{
  "url": "https://example.com/page",
  "title": "Page Title",
  "description": "Meta description text",
  "links": ["https://example.com/other", ...]
}
```

## Remarks
- The crawler respects a **1‑second delay** to avoid hammering the target site. Increase `--delay` for larger sites.
- Only **same‑domain** links are followed; external URLs are listed but not crawled.
- For very large sites consider expanding the script with a database or multi‑process workers.
